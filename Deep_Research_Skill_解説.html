Deep Research Skill

Claude Codeを「本格リサーチエージェント」として動かす拡張機能の仕組みと使い方

元資料：199-biotechnologies/claude-deep-research-skill | 作成日：2026-05-17

このSkillが解決する問題

普通のAIチャットの限界

ChatGPTやClaudeに質問すると、すぐに答えが返ってきます。でもその答えには大きな弱点があります。

  • AIが知っているのは学習した時点までの情報だけ
  • 情報が正しいかどうかの検証をしてくれない
  • 引用元（ソース）が提示されない
  • 仕事の意思決定に使える「信頼できるレポート」とは程遠い

Deep Research Skillの解決策

Claude Code（ターミナルで動くClaudeのツール）に追加できる拡張機能（Skill）で、Claudeを「本格的なリサーチアナリスト」として動かす仕組みです。

  • Web検索を何度も自動で実行して情報を集める
  • 複数のソースで裏取り（三角測量）する
  • 自分で批判的にチェックし、穴があれば追加検索する
  • 引用付きの本格レポートを自動生成する

使い方フロー（5ステップ）

STEP 0

使う前に必要なものを揃える

必要なもの備考
macOS または LinuxWindowsはWSLが必要
ターミナルmacOSなら「ターミナル.app」が最初から入っている
Claude CodeAnthropicの公式CLIツール
gitほとんどのMacに最初から入っている
Claude Codeがまだない場合は、ターミナルで以下を実行します
npm install -g @anthropic-ai/claude-code
STEP 1

Skillをインストールする（1回だけ）

ターミナルで以下の1行をコピー＆ペーストして Enter を押します。

# Claude Codeのスキルフォルダにダウンロード
git clone https://github.com/199-biotechnologies/claude-deep-research-skill.git ~/.claude/skill

完了メッセージが出たらインストール終了です。これだけです。

オプション：search-cli

複数の検索エンジン（Brave・Serper・Exa・Jina・Firecrawl）を使いたい場合は追加でインストールできます。ただし各サービスのAPIキー取得が必要なので、まずは入れなくてもOKです。

brew tap 199-biotechnologies/tap && brew install search-cli
search config set keys.brave YOUR_KEY
STEP 2

Claude Code を起動する

ターミナルで以下を入力して Claude Code を起動します。

claude

チャット画面のようなインターフェースが表示されれば準備完了です。

初回起動時はAnthropicアカウントへのログインが求められます。案内に従ってブラウザで認証してください。
STEP 3

リサーチを指示する

Claude Codeのチャット欄に、以下の形式で入力します。

# 基本形
deep research on [調べたいテーマ]

# モード指定あり
deep research in ultradeep mode: [テーマ]

リサーチモードの選び方

モード所要時間フェーズ数こんなときに使う
Quick2〜5分3ざっくり知りたい・最初の下調べ
Standard5〜10分6迷ったらこれ。ほとんどの用途に対応
Deep10〜20分8重要な意思決定・複雑なテーマ
UltraDeep20〜45分8以上完璧なレポートが必要なとき
STEP 4

自動実行（待つだけ）

あとはClaudeが自動で調査を進めます。画面に進捗が流れてきますが、何もしなくて大丈夫です。内部では以下の8フェーズが順番に動いています。

内部で動く8フェーズ

1 Scope 何を調べるか整理する
2 Plan 調査方針を立てる
3 Retrieve 5〜10件の検索を同時実行
4 Triangulate 複数ソースで照合・信頼性評価
5 Outline レポートの骨格を見直す
6 Synthesize 情報を統合して文章化
7 Critique 3つの視点で批判的チェック
8 Refine 修正して最終パッケージ化

フェーズ7「Critique」の仕組み

3つの異なる視点でレポートをチェックします：

  • 懐疑的な実務家 — 「本当にこれで実務に使えるのか？」
  • 反論を探す批評家 — 「この結論を否定する情報はないか？」
  • 実装エンジニア — 「具体的にどう実装するのか不明確な点は？」

重大な抜け穴が見つかると自動でフェーズ3に戻って追加検索します。
これが他のAIリサーチとの最大の違いです。

STEP 5

レポートを確認する

完了すると、以下のフォルダにレポートが自動保存されます。

~/Documents/[トピック名]_Research_[日付]/

同じ内容が3つの形式で出力されます：

形式特徴
Markdown（.md）テキストの正式データ。編集・加工しやすい
HTMLMcKinsey風デザイン。完了後に自動でブラウザが開く
PDF印刷・共有向け。WeasyPrintで整形済み
HTMLファイルが自動でブラウザに開くので、まずそこから確認するのがいちばん見やすいです

品質管理の仕組み

出力されるレポートには厳格な品質基準が設定されています。

最低ソース数 10以上
主要主張の根拠 3つ以上
サマリー文量 200〜400語
各セクション 600〜2,000語
散文比率 80%以上
参考文献 URL付き完全書誌

自動検証スクリプト

完成後に以下の2つのスクリプトが自動で走ります：

  • validate_report.py — 9項目の構造チェック
  • verify_citations.py — URL実在確認＆ハルシネーション（嘘情報）検出

問題があれば自動修正を最大3回試みます。

検索ツールの優先順位

ツール役割
search-cliメイン検索。インストール済みならすべての検索はここ経由
WebSearchsearch-cliが使えない・レート制限時のフォールバック（代替手段）
Exa MCPオプション。セマンティック検索（意味で検索）を追加したい場合

アーキテクチャ（ファイル構成）

Skill全体は以下のようなフォルダ構成になっています。

deep-research/
├── SKILL.md ← エントリーポイント（約100行）
├── reference/
│   ├── methodology.md ← 8フェーズの詳細定義
│   ├── report-assembly.md ← レポート組み立てルール
│   ├── quality-gates.md ← 品質基準の定義
│   ├── html-generation.md ← HTML生成ルール
│   └── continuation.md ← 中断時の再開方法
├── templates/
│   ├── report_template.md ← Markdownテンプレート
│   └── mckinsey_report_template.html ← HTMLテンプレート
├── scripts/
│   ├── validate_report.py ← 9項目の構造チェック
│   ├── verify_citations.py ← URL実在確認・嘘検出
│   ├── source_evaluator.py ← ソース信頼性スコアリング
│   ├── citation_manager.py ← 引用管理
│   ├── md_to_html.py ← Markdown→HTML変換
│   └── research_engine.py ← コア制御エンジン
└── tests/
    └── fixtures/

まとめ

このSkillの本質は、「Claudeを1つのチャットボットとして使うのではなく、複数の役割を持つエージェントチームとして動かす」ことです。

調査担当・批評担当・検証担当がそれぞれ動き、最終的に引用付きの本格レポートが完成する構造になっています。

こんな場面で威力を発揮します：

  • 競合分析のレポート作成
  • 新技術・新サービスの調査
  • 意思決定のための根拠収集
  • 市場動向のリサーチ

普通にClaudeに聞くだけと比べて、出力の信頼性が格段に上がります。
